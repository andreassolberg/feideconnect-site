Walk through of Feide Connect for a developer

In this walkthrough we will show you all the details of Feide Connect in practice from a developers perspective.

We will first demonstrate how to use Feide Connect to protect an API, and then connect an application to obtain access to this API.

For the use case we will be using the UNINETT Foodle service – a simple survey and poll web tool.

The Feide Connect API Gatekeeper

Protecting an API with an OAuth Provider is not that complex. However, you also have to add federated authentication to the provider. And then a easy to use self service portal for third party clients to register – and that would need to be federated as well. And add policy management and workflow into the mix. And you want todo this for all your APIs spanning across your organization. You will need to build something like Feide Connect, or simply let Feide Connect help you. Here's how.

We already have an API, and we want to implement handling of trusted requests from Feide Connect.

A trusted requeset from Feide Connect, is passed on from an authenticated client and an authenticated user of the Feide Connect platform. It looks like this:
