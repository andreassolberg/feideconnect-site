Authentication as an end user

Before the educational institution you belong to is successfully connected to the Connect platform, you may choose to login using OpenIdP in the login provider selection interface.

Getting started with Service Providers / Applications

As an service provider, you may register a client which implement OAuth Consumer functionality.

Learn about OAuth before moving on.

Register a client using the Developer Dashboard. When registering a client, you need to know the OAuth redirect URI endpoint of your application.

HTTP or HTTPS The Connect pilot will allow you to register Redirect URIs that are not secured with HTTPs, although not reccomended. A production enviornment with Connect, would probably forbid registering unsecured endpoints.
  • Developer Dashboard
  • Getting started with OAuth using UNINETT Connect (not entirely up to date)

When you have successfully registered a client, you need to know the OAuth endpoint of Connect, in order to configure your application:

Connect OAuth Provider Authorization endpoint
https://auth.feideconnect.no/oauth/authorization
Connect OAuth Provider Token endpoint
https://auth.feideconnect.no/oauth/token

OAuth configuration JSON endpoint

User information

With the obtained OAuth token, a request to the user info endpoint will give information about the authenticated user.

The set of userdata you obtain, will depend on which scopes you have requested. Scopes is managed through the developer dashboard.

To get the basic information about a user, you might start with requesting the userinfo, userinfo-photo, userinfo-mail, userinfo-feide scopes.

Example

{% highlight http %} GET https://auth.dev.feideconnect.no/userinfo HTTP/1.1 Host: auth.feideconnect.no Authorization: Bearer 6c443f1e-7052-432c-1234-abc2644de4ab {% endhighlight %} {% highlight http %} HTTP/1.1 200 OK Content-Type:application/json; charset=utf-8 { "user": { "userid": "76a7a061-3c55-430d-8ee0-6f82ec42501f", "userid_sec": [ "feide:andreas@uninett.no" ], "name": "Andreas Ã…kre Solberg", "email": "andreas.solberg@uninett.no", "profilephoto": "p:a3019954-902f-45a3-b4ee-bca7b48ab507" }, "audience": "cf00cea5-fc20-4c90-8515-144159322bed" } {% endhighlight %}

The user profile photo is available on the URL including the profilephoto value from the userinfo endpoint: 

https://auth.feideconnect.no/user/media/p:a3019954-902f-45a3-b4ee-bca7b48ab507

Most Connect APIs support CORS, including the userinfo API, to use with client side applications.
